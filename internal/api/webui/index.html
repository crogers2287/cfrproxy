Provider registry

Model map

Rewrite fixed harness model names to any provider/model — e.g. map claude-opus*, claude-sonnet*, claude-haiku* so Claude Code's built-in /model presets become switchable cfrproxy slots. Patterns: exact name or trailing *. Unmapped unknown names route to the top provider's default model.

Auto router

Send model auto from any harness and a small classifier model buckets the request, then delegates to the model you map per task. The chosen route shows in Live Traces.

Consensus round table (MCP)

Define a panel of agent profiles — each a persona pinned to a different model behind the router — and let them deliberate a question together. cfrproxy exposes this as an MCP server, so any MCP-capable agent (Claude Code, etc.) can convene the panel as a tool.

1 · Fan out
every enabled profile answers independently, in parallel
2 · Cross-critique
each panelist sees the others and revises (round 2)
3 · Synthesize
the moderator merges into consensus + dissent
claude mcp add roundtable -- cfrproxy mcp
Tools exposed: roundtable (full panel + synthesis) · consult (ask one profile) · list_profiles. Every voice routes through cfrproxy, so panelists can mix any providers — a local model, a Claude subscription, a Grok model — at the same table.

Agent profiles (panelists)

Each profile is one voice on the panel: a persona (system prompt) pinned to a model. Disabled profiles sit out the round table.

Round table settings

Compress the shared context once before fan-out (summarizes the question's context so each panelist gets the short version, not the full payload × N). Uses the summarizer from the Context compression section — works even if global compression is off.

Context compression

When a request exceeds the threshold, older turns are summarized by the summarizer model and replaced with one compressed block (recent turns stay verbatim; summaries cached per conversation prefix). Trace note shows compressed ~A→~B tok.
